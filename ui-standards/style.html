Typography

Font family: Inter / system-ui stack. All sizes use rem units.

Heading 1 — 1.5rem / 700

h1 { font-size: 1.5rem; font-weight: 700 }

Heading 2 — 1.25rem / 700

h2 { font-size: 1.25rem; font-weight: 700 }

Heading 3 — 1rem / 600

h3 { font-size: 1rem; font-weight: 600 }

Body text — 0.875rem / 400. The standard paragraph size for content areas. This is what most UI text uses throughout the application, including table cells, descriptions, and form labels.

body { font-size: 0.875rem }

Settings subtext — 0.9rem, var(--text-secondary). Used for descriptive help text below settings labels.

.settings-subtext { font-size: 0.9rem; color: var(--text-secondary) }

Monospace — price columns, code references: $1,234.56

font-family: "SF Mono", "Fira Code", monospace

Buttons

Current Standard

All buttons use rounded pill shape (border-radius: 999px). This is the standard as of v3.33.16. Apply to all new UI work.

Modal Footer Buttons (Primary Pattern)

Used in all modal action bars — compact, pill-shaped, bold weight. Scope the pill CSS to your modal's action container.

.modal-footer .btn { border-radius: 999px; padding: 0.35rem 0.9rem; font-size: 0.78rem; font-weight: 700 }

Issues with this pattern

  • Square corners look heavy and Bootstrap-like
  • Large min-height (2.75rem) wastes vertical space in compact UIs
  • Gradient shine hover effect (.btn::before) looks dated
  • Inconsistent with the pill pattern used in modal footers since v3.33.16

Modal Headers

Current Standard

Glass-style header with var(--bg-secondary) background and subtle border-bottom. No gradient. Pill buttons in the footer. This is the standard as of v3.33.16.

Glass Header Modal

Modal Title

×

Glass-morphism modal with subtle background. Header uses var(--bg-secondary) — no gradient banner. Clean, professional look that works across all four themes.

.modal-header { background: var(--bg-secondary); border-bottom: 1px solid var(--border) }

CSS Pattern

/* Modal sizing — glass-morphism inherited from .modal-content */
.my-modal-content {
  max-width: 600px;
  padding: 0;
  display: flex;
  flex-direction: column;
  overflow: hidden;
}

/* Glass header (CURRENT standard — NOT gradient) */
#myModal .modal-header {
  background: var(--bg-secondary);
  color: var(--text-primary);
  padding: var(--spacing) var(--spacing-xl);
  border-bottom: 1px solid var(--border);
  box-shadow: none;
}
#myModal .modal-header h2 {
  color: var(--text-primary);
  font-size: 0.92rem;
  font-weight: 700;
  margin: 0;
}

/* Footer — pill buttons */
.my-modal-actions {
  display: flex;
  justify-content: flex-end;
  gap: 0.5rem;
  padding: var(--spacing);
  border-top: 1px solid var(--border);
}

Modal Headers

Deprecated

The old gradient banner modal header. This style was widely used pre-v3.33.16 but looks heavy and is strongly associated with "AI slop." Do not use in new code. Migrate to glass header when editing existing modals.

Gradient Banner Header

Modal Title

×

This is the old gradient banner style. It looks heavy and dated. Everyone associates it with AI-generated code.

Issues with this pattern

  • Gradient banner is the #1 visual tell of AI-generated UI
  • White text on gradient has contrast issues in light themes
  • Looks heavy and dated compared to modern glass-morphism
  • Square buttons in the footer look inconsistent with pill buttons elsewhere
Replaced by: Glass header (var(--bg-secondary) + border-bottom) — see Modal Headers (Current)

Migration Checklist

When editing any existing modal or component, check these items and migrate deprecated patterns to current ones while you're there.

Check Deprecated Current
Modal header linear-gradient(135deg, ...) background: var(--bg-secondary)
Action buttons shape border-radius: var(--radius) (8px square) border-radius: 999px (pill)
Button class syntax .btn-primary, .btn-secondary (Bootstrap) .btn, .btn.secondary (space-separated)
Settings toggle <input type="checkbox"> .chip-sort-toggle
Modal open/close modal.style.display = 'none' closeModalById('id')
Color values Hardcoded hex (#3b82f6) CSS var (var(--primary))
